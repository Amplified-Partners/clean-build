# Swift Code Style Guidelines

## Naming Conventions
- Use camelCase for variables, functions, properties: `currentStreak`, `loadTasks()`
- Use PascalCase for types (structs, classes, enums): `StorageService`, `DailyLog`, `StreakData`
- Use CONSTANT_CASE for file-scoped constants

## Type Safety
- Always provide explicit type annotations for ambiguous expressions
- Use modern Swift syntax: `let`, `const`, guard statements
- Avoid force unwrapping (`!`) except in unavoidable cases; prefer `??` or `guard let`

## Async & Error Handling
- Prefer `try?` with fallback values over bare try-catch when appropriate
- Always validate file I/O operations; provide sensible defaults for missing files
- Use guard statements for early returns on validation failure

## Documentation
- Add MARK comments to separate logical sections: `// MARK: - Tasks`
- Include brief doc comments for public methods
- Debug print statements must use descriptive prefixes: `🔍 DEBUG:`, `❌ DEBUG:`, `✅ Success:`

## Code Organization
- Keep methods grouped by concern (Tasks, Interactions, Streak Data, Logs)
- Single-file implementations preferred unless >500 lines of code
- Embed CSS and JavaScript in HTML; avoid separate files unless explicit requirement
