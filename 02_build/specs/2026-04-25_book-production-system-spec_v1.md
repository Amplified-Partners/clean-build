## Part 2: Visual Design System
### Cutting-Edge, Zero-Bloat Information Design

Based on Tufte principles + cognitive load research + 2024-2025 learning science.

---

### The Data-Ink Ratio Principle (Tufte)

**Core Rule:** Maximise ink that represents actual data. Minimise everything else.

```
Data-ink ratio = Data-ink / Total ink used

Target: As close to 1.0 as possible
```

**What to ELIMINATE:**
- Chartjunk (decorative elements, 3D effects, shadows)
- Moiré vibration (distracting patterns)
- Heavy gridlines
- Redundant labels
- Non-functional colours
- Decorative borders/frames

**What to KEEP:**
- Numbers themselves
- Meaningful trend lines
- Essential axis labels (minimal)
- Clear typography

---

### Visual Types for the Book

| Visual Type | Use Case | Tufte Compliance | Implementation |
|-------------|----------|------------------|----------------|
| **Sparklines** | Inline trends (word-sized) | Highest data-ink | Embedded in text |
| **Small Multiples** | Comparing frameworks | High density | Grid layouts |
| **Tables** | Statistics, comparisons | Minimal lines | Key data only |
| **Process Flows** | Systems, workflows | Clean arrows | No decoration |
| **Before/After** | Transformations | Side-by-side | Stark contrast |
| **Pudding Maps** | Cross-expert connections | Novel format | Graph-style |

---

### Cognitive Load Management

**Research Finding (2024-2025):**
- Modular design reduces extraneous cognitive load by **31%** (p<.001)
- Working memory has **two channels**: visual + auditory
- Prior knowledge moderates learning effectiveness
- Emotional design (warm colours, approachable faces) increases retention

**Book Design Implications:**

1. **Chunk aggressively** - No section longer than can be absorbed in 5 minutes
2. **Visual + verbal together** - Every concept has both text AND diagram
3. **Progressive disclosure** - Simple first, detail on demand
4. **Modular structure** - Each chapter standalone, but connected
5. **Emotional anchors** - Stories, humour, relatable examples

---

### Typography & Layout Science

| Element | Recommendation | Evidence |
|---------|----------------|----------|
| Line length | 45-75 characters | Reading comprehension research |
| Font size (digital) | 16-18px minimum | Accessibility + retention |
| Paragraph spacing | 1.5-2x line height | Reduced cognitive load |
| Heading hierarchy | 3 levels max | Information processing limits |
| Whitespace | Generous margins | Reduces perceived complexity |
| Contrast ratio | 7:1 minimum | WCAG + readability |

```

│     • "Did you know..." or "What if..."                        │
│                                                                 │
│  2. PROBLEM (2 minutes)                                        │
│     • The pain SMB owners feel                                 │
│     • Specific, relatable scenario                             │
│     • Validate without patronising                             │
│                                                                 │
│  3. GIANT'S WISDOM (3 minutes)                                 │
│     • Core principle from expert(s)                            │
│     • Statistics/results (attributed)                          │
│     • Original insight, not rehash                             │
│                                                                 │
│  4. PUDDING (2 minutes)                                        │
│     • The unexpected connection                                │
│     • "Here's what X and Y have in common..."                  │
│     • The novel insight that's YOURS                           │
│                                                                 │
│  5. IMPLEMENTATION (3 minutes)                                 │
│     • Concrete action steps                                    │
│     • "Today, do this one thing..."                            │
│     • Low-friction entry point                                 │
│                                                                 │
│  6. VISUAL (embedded throughout)                               │
│     • One key diagram/table                                    │
│     • Tufte-compliant                                          │
│     • Reinforces core concept                                  │
│                                                                 │
│  7. BRIDGE (30 seconds)                                        │
│     • Connection to next chapter                               │
│     • "Now that you know X, let's explore..."                  │
│                                                                 │
│  TOTAL: ~10 minutes reading time                               │
```

### Publisher Script (Your Mac)

```python
#!/usr/bin/env python3
"""
Book Publisher - Orchestrates the 4 Railway agents
to produce chapter drafts from the database.

Run this on your Mac after ingestion + pudding pass complete.
"""

import requests
import json
from pathlib import Path

# Configuration
AGENTS = {
    "researcher": "https://your-railway-url/agent-1",
    "writer": "https://your-railway-url/agent-2", 
    "editor": "https://your-railway-url/agent-3",
    "pudding": "https://your-railway-url/agent-4"
}

OUTPUT_DIR = Path.home() / "vault" / "work-covered-ai" / "book-output"

def produce_chapter(chapter_num: int, section: str, topic: str):
    """
    Full pipeline for one chapter.
    """
    print(f"📖 Starting Chapter {chapter_num}: {topic}")
    
    # Step 1: Research
    print("  1️⃣ Researching...")
    research = requests.post(
        AGENTS["researcher"],
        json={"section": section, "topic": topic, "chapter": chapter_num}
    ).json()
    
    # Step 2: Get Puddings
    print("  2️⃣ Finding puddings...")
    puddings = requests.post(
        AGENTS["pudding"],
        json={"concepts": research["key_concepts"]}
    ).json()
    
    # Step 3: Write
    print("  3️⃣ Writing draft...")
    draft = requests.post(
        AGENTS["writer"],
        json={
            "research": research,
            "puddings": puddings,
            "chapter": chapter_num,
            "topic": topic
        }
    ).json()
    
    # Step 4: Edit
    print("  4️⃣ Editing...")
    final = requests.post(
        AGENTS["editor"],
        json={"draft": draft["markdown"]}
    ).json()
    
    # Step 5: Save
    output_path = OUTPUT_DIR / f"chapter-{chapter_num:02d}-{section}.md"
    output_path.write_text(final["markdown"])
    print(f"  ✅ Saved to {output_path}")
    
    return final

if __name__ == "__main__":
    # Book outline
    chapters = [
        (1, "thesis", "The Myth of More Hours"),
        (2, "market", "Information Overload Disease"),
        (3, "market", "Why Good Businesses Fail"),
        # ... continue for all 20 chapters
    ]
    
    for num, section, topic in chapters:
        produce_chapter(num, section, topic)
        input("Press Enter to continue to next chapter...")
```
