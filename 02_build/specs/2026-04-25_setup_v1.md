# AI Orchestrator - Setup & Usage Guide

## Quick Start

### 1. Install Dependencies

```bash
cd ~/ai-orchestrator
python3 -m venv venv
source venv/bin/activate  # On Windows: venv\Scripts\activate
pip install -r requirements.txt
```

### 2. Configure Environment

```bash
cp .env.example .env
# Edit .env with your API keys
```

Required environment variables:
```env
# Optional - for cloud models
ANTHROPIC_API_KEY=your_key_here
OPENAI_API_KEY=your_key_here

# Optional - for local models (defaults to localhost)
OLLAMA_BASE_URL=http://localhost:11434

# Database (optional, defaults to SQLite)
DATABASE_URL=sqlite+aiosqlite:///./ai_orchestrator.db
```

### 3. Initialize Database

```bash
python main.py --init-db
```

### 4. Run Your First Task

```bash
# Simple query (will use local model if available)
python main.py --task "What is the capital of France?"

# Force cloud model with high quality
python main.py --task "Explain quantum computing" --model cloud --quality high

# Code generation
python main.py --task "Write a Python function to calculate fibonacci" --task-type code

# With cost constraint
python main.py --task "Generate a story" --max-cost 0.01
```

## Usage Examples

### Basic Usage

```bash
# Auto-select best model
python main.py --task "Your task here"

# Prefer local models
python main.py --task "Your task here" --model local

# Prefer cloud models
python main.py --task "Your task here" --model cloud

# High quality (uses best models)
python main.py --task "Your task here" --quality high

# Low cost (uses cheapest models)
python main.py --task "Your task here" --quality low
```

### Advanced Options

```bash
# Specify task type for better routing
python main.py --task "Your task" --task-type code

# Set complexity manually
python main.py --task "Your task" --complexity complex

# Limit cost per request
python main.py --task "Your task" --max-cost 0.05

# Stream response in real-time
python main.py --task "Write a story" --stream

# Show detailed metrics
python main.py --task "Your task" --show-metrics

# JSON output
python main.py --task "Your task" --json

# Verbose logging
python main.py --task "Your task" --verbose
```

### View Statistics

```bash
# Show usage statistics
python main.py --stats
```

## Architecture

The system consists of several key components:

1. **TaskAnalyzer** (`core/task_analyzer.py`)
   - Analyzes tasks to determine type, complexity, and requirements
   - Provides routing hints based on task characteristics

2. **Router** (`core/router.py`)
   - Selects optimal model based on task profile and routing rules
   - Considers cost constraints and model availability
   - Provides fallback options

3. **Database** (`core/database.py`)
   - SQLAlchemy ORM for async database operations
   - Tracks tasks, executions, costs, and performance metrics

4. **CostTracker** (`core/cost_tracker.py`)
   - Real-time cost tracking and budget management
   - Provides cost estimates and optimization recommendations

5. **Orchestrator** (`core/orchestrator.py`)
   - Main coordination engine
   - Ties all components together
   - Manages complete task lifecycle

## Configuration Files

### `config/models.yaml`
Defines available models and their capabilities.

### `config/routing_rules.yaml`
Defines routing logic based on task characteristics.

### `config/costs.yaml`
Defines cost per model (per 1M tokens).

## Graceful Degradation

The system works with or without API keys:

- **No API keys**: Uses local Ollama models only
- **Partial keys**: Uses available providers and falls back to local
- **All keys**: Full routing across all providers

## Database Schema

The system tracks:
- **tasks**: All task requests
- **executions**: Model execution attempts
- **cost_summary**: Aggregated daily costs
- **routing_decisions**: Routing logic decisions
- **model_performance**: Performance metrics by model

## Cost Management

The system includes built-in cost management:

- Daily budget limits (default: $50/day)
- Per-request cost limits (default: $1/request)
- Real-time cost tracking
- Budget alerts at 50% and 80%
- Cost optimization recommendations

## Troubleshooting

### "No module named 'core'"
Make sure you're in the ai-orchestrator directory and have activated the virtual environment.

### "ANTHROPIC_API_KEY not set"
Either set the API key in `.env` or use `--model local` to force local models.

### "Ollama not available"
Make sure Ollama is running: `ollama serve`

### Database errors
Reinitialize the database: `python main.py --init-db`

## Production Deployment

For production use:

1. Use PostgreSQL instead of SQLite:
   ```env
   DATABASE_URL=postgresql+asyncpg://user:pass@localhost/ai_orchestrator
   ```

2. Set appropriate budget limits in `config/costs.yaml`

3. Enable logging to file:
   ```python
   logging.basicConfig(
       level=logging.INFO,
       handlers=[
           logging.FileHandler('ai_orchestrator.log'),
           logging.StreamHandler()
       ]
   )
   ```

4. Consider adding authentication and rate limiting

5. Monitor costs regularly using `--stats`

### Adding New Models

1. Add model configuration to `config/models.yaml`
2. Add cost information to `config/costs.yaml`
3. Update routing rules in `config/routing_rules.yaml` if needed
4. Test with `pytest tests/ -v`

### Adding New Routing Rules

Edit `config/routing_rules.yaml` and add a new rule:

```yaml
- name: my_custom_rule
  conditions:
    task_type: custom
    complexity: medium
  model_preference:
    - provider: anthropic
      model: claude-3-5-sonnet-20241022
```
