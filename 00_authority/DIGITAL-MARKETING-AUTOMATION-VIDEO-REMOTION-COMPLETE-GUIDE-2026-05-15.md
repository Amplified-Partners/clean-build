### Hyper-Local Strategy

**Geographic targeting:**
- Trades: 5-10 mile radius
- Retail: 1-3 mile radius
- Services: City/region

**Why it works:** A plumber in Newcastle doesn't compete with a plumber in Manchester. Template reuse without conflict. Lower competition (not national). Higher relevance (local trust). Lower ad costs (geo-targeted). Better conversion (people prefer local).

### The 16-Point Friction Taxonomy

From the Value Creation department — the systematic framework for identifying what to automate:

**Process:** Map client operations against all 16 universal friction point categories -> Score severity (impact x frequency) -> Estimate time cost per friction point (hours/week) -> Calculate ROI of elimination -> Prioritise by ROI -> Build automation roadmap.

**Automation target:** Technical potential is 60-70% of operations. Realised capture at SMB industry baseline is 35-45%. Moving a client from 0% to ceiling is a 12-24 month project measured by *their* data.

## 5. Remotion Pipeline

> Source: Remotion Video Pipeline [77-cite-4]
> `03_shadow/specifications/2026-04_amplified-marketing-engine_v2.md` (§Remotion Video Pipeline)
> `01_truth/schemas/departments/dept-marketing_v1.md` (§Amplified Video V5 Spec, §Video Production Strategy)

### What Remotion Is

[Remotion](https://github.com/remotion-dev/remotion) (34.4K GitHub stars) creates MP4 videos programmatically using React. As of January 2026, Remotion Agent Skills allow AI-to-video generation via MCP.

**Why it matters:** Programmatic generation means same template, different data, hundreds of variations at near-zero marginal cost. This is the bridge from manual video production (Section 4) to fully automated, data-driven video at scale.

### Architecture

```
Research brief
    |
    v
Script Node --> segment-adapted scripts from Claude (by trade, archetype, channel)
    |
    v
Asset Node --> source images + generate voiceover (ElevenLabs or local TTS)
    |
    v
Composition Node --> Remotion React composition from branded templates
    |
    v
Render Node --> server-side MP4 render on Beast CPU (96 threads)
    |
    v
Evaluation Node --> synthetic evaluator panel scores the video
    |
    v
Human review --> approve/reject
    |
    v
Distribute --> TikTok, Instagram, YouTube, LinkedIn
```

### Beast Hardware Constraint

**Beast = Hetzner AX162-R** — AMD EPYC 9454P, 48-core / 96-thread, 251 GB RAM. **No GPU.** Remotion renders on CPU. Rendering and Ollama inference share the same compute pool. Benchmark render queue depth and concurrency vs inference load before committing to scale.

### Dependencies

| Dependency | Status | Notes |
|-----------|--------|-------|
| Node.js + React on Beast | Required | For Remotion runtime |
| Branded video templates | `[DECISION REQUIRED]` | Design work needed — brand assets |
| Voiceover | `[LOGIC TO BE CONFIRMED]` | ElevenLabs API (~£5-22/mo) or local TTS |
| CPU capacity planning | Required | Rendering vs Ollama inference — shared pool |
| Synthetic evaluator extension | Required | Video-specific dimensions (visual coherence, pacing, voiceover quality) |

### Economics

| Item | Cost |
|------|------|
| Remotion rendering | £0 (self-hosted, open-source) |
| ElevenLabs voiceover | ~£5-22/month `[LOGIC TO BE CONFIRMED]` |
| Beast compute | Already provisioned |

### Implementation Plan (from Marketing Engine Spec, Phase 3)

**Weeks 7-12:** Single avatar, single template, single channel prototype.

1. Install Remotion on Beast. Benchmark CPU rendering vs Ollama inference load.
2. Design one branded video template (Bob the plumber, LinkedIn).

### The Live Pipeline (6 Stages)

**Pipeline:** `research -> generate -> evaluate -> queue -> learn` (sequential cron, 4am UTC, port 8000, 3-tier auth).

#### Stage 1: Research

- **Engine:** SearXNG (243+ sources, Beast-first search — no rate limits, 10Gbps)
- **Code:** `agents/research_agent.py` (461 lines)
- **Grounding:** Vector store + graph via `amplified-knowledge-mcp`
- **Output:** Research briefs with attributed sources and confidence bands (PROVEN / VALID / HYPOTHESIS)
- **Gate:** Every brief must have >=1 attributed source. No `[SOURCE REQUIRED]` briefs pass.

#### Stage 2: Content Generation

- **Code:** `agents/content_agent.py` (369 lines), `agents/content_atomizer.py` (317 lines)
- **Process:** Pillar content -> 5 platform variants via LiteLLM
- **Adapters:** 40 campaigns: 10 trades x 4 archetypes (Bob, Sheila, Dave, Russell)
- **50+ UK business types** with trade-specific pain points as questions (permission-based)
- **Transparent prompts:** Amplified speaks as itself, not pretending to be tradespeople
- **Gate:** Every piece cites >=1 source. Passes transparent prompt check.

#### Stage 3: Synthetic Evaluation

- **Code:** `synthetic_evaluator.py` (546 lines)
- **8 personas built:** Bob, Lisa, Marcus, Sarah, Technical Tom, Skeptical Sam, Budget Brian, Growth Gina
- **3 currently running:** Bob/Lisa/Marcus
- **5 dimensions:** Relevance, honesty, clarity, actionability, attribution (1-10 scale)
- **Quality routing:** >=7/10 -> auto-queue for human review. 6/10 -> revision queue. <6/10 -> reject with feedback to Kaizen.
- **Gate:** No content reaches human review without passing both synthetic evaluation and automated checks.

#### Stage 4: Human Review

- Ewan (or delegated reviewer) reviews via Command Centre + API
- Approve / reject / edit. Edits feed back to Kaizen preferences.
- **Telegram gate** for publish approval.
- **Gate:** Human approval is mandatory. No automated publishing without explicit sign-off. Non-negotiable.

#### Stage 5: Distribution

- **Code:** `agents/publishing_agent.py` (256 lines), `platform_adapters.py` (431 lines)
- **Built adapters:** LinkedIn (OAuth), Substack, Twitter/X, Email (Brevo), Blog
- **Timing:** Platform-optimal UK timing
- **Reddit:** Never automated — human-in-the-loop posting only

#### Stage 6: Feedback and Kaizen

- **Code:** `kaizen.py` (367 lines) + `kaizen-optimizer` container + Cove Kaizen workflow (1,320 lines on Temporal)
- Self-improving, auto-applying with Layer 0 locked and max 3 auto-applies/cycle
- Retrospective -> insights -> suggestions -> auto-apply
