## Document Maintenance

### When to Update

**Update Infranodus Doc When:**
- New research on GraphRAG theory emerges
- Competitors release significant features
- Your learnings contradict initial assumptions

**Update Architecture Doc When:**
- You complete a phase and learn implementation details
- Technology changes (new Neo4j features, etc)
- Costs shift significantly

**Update Master Knowledge Doc When:**
- Financial projections prove inaccurate
- You hit go/no-go milestones
- Market conditions change
- New competitive threats emerge

### How to Update

Add a "LEARNED" section at end of each document:
```
## LEARNED (Real Experience vs Plan)

**Date:** [When you realized this]
**Learning:** [What you discovered]
**Impact:** [How it changes the strategy]
**Updated Section:** [Which section needs revision]
**Action:** [What to do differently]
```

Example:
```
## LEARNED

**Date:** March 2026 (after Phase 1)
**Learning:** Dental practices respond better to £150/mo than £200/mo
**Impact:** Unit economics still work at £150, but acquisition much faster
**Updated Section:** Master Knowledge Doc, Section 8 (Pricing)
**Action:** Lower Starter tier to £150, test market response
```
