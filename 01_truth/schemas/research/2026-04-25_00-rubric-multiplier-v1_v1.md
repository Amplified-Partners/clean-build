# PART 2: ATTRIBUTION RUBRIC (The Foundation)

Nothing else matters if we can't trace where it came from.

## 2.1 Source Attribution

| Score | Level | Definition | Multiplier Effect |
|-------|-------|------------|-------------------|
| 0 | Unknown | No traceable origin | Cannot automate, human review only |
| 1 | Inferred | Source guessed from context | Limited automation, flag for verification |
| 2 | Stated | Source named in content | Can cite, can automate with caveats |
| 3 | Verified | Source confirmed, linked to original | Full automation, full citation, full trust |

**Rule:** Score 0-1 = AMBER bucket. Score 2-3 = GREEN bucket.

## 2.2 Temporal Attribution

| Score | Level | Definition | Multiplier Effect |
|-------|-------|------------|-------------------|
| 0 | Unknown | No date information | Cannot assess freshness |
| 1 | Approximate | Year or quarter known | Can filter by era |
| 2 | Dated | Specific date known | Can track evolution |
| 3 | Timestamped | Exact datetime with context | Full temporal reasoning |

**Rule:** Evergreen content (principles) can score 3 even without recent dates.

## 2.3 Method Attribution

| Score | Level | Definition | Multiplier Effect |
|-------|-------|------------|-------------------|
| 0 | Unknown | How was this gathered? | Cannot weight appropriately |
| 1 | Stated | Method named (interview, observation, etc.) | Can categorize |
| 2 | Documented | Method + conditions recorded | Can replicate |
| 3 | Systematic | Follows defined methodology | Can automate collection |

**Attribution Total: /9**
- 7-9 = Full automation enabled
- 4-6 = Partial automation, human oversight
- 0-3 = Human review required

# PART 5: DECISION LOGIC RUBRIC (The Automation Layer)

The best pieces encode IF-THEN logic that agents can execute.

## 5.1 Decision Logic Presence

| Score | Level | Definition | Multiplier Effect |
|-------|-------|------------|-------------------|
| 0 | None | No actionable logic | Information only |
| 1 | Implicit | Logic can be inferred | Human interpretation needed |
| 2 | Stated | IF-THEN logic written out | Can be coded |
| 3 | Structured | YAML/JSON decision tree | Direct automation |

## 5.2 Decision Logic Format

When logic exists, it should follow:

```
TRIGGER: [situation that activates this]
CONDITION: [IF this is true]
ACTION: [THEN do this]
OUTCOME: [Expected result]
FALLBACK: [IF action fails, do this]
```

**Example:**
```
TRIGGER: Client mentions pricing uncertainty
CONDITION: IF no existing pricing principle documented
ACTION: Apply Kennedy DK_003 (Price on Value)
OUTCOME: Value-anchored price recommendation
FALLBACK: Escalate to human for custom pricing
```

# PART 7: THE MULTIPLIER FORMULA

## 7.1 Total Multiplier Score

```
MULTIPLIER = (Attribution × Authority × Dimensions × Logic × Validation)
```

Each factor is 0-3 (or 0-4 for validation).

**Maximum possible:** 3 × 3 × 3 × 3 × 4 = **324**

**Interpretation:**
- 200+ = Gold (maximum automation, maximum trust)
- 100-199 = Silver (partial automation, good trust)
- 50-99 = Bronze (limited automation, human oversight)
- <50 = Raw (requires processing before use)

## 7.2 Why Multiplication, Not Addition

Addition: 3 + 3 + 3 + 3 + 4 = 16 (linear)
Multiplication: 3 × 3 × 3 × 3 × 4 = 324 (exponential)

**A zero in ANY dimension zeros the whole thing.**

- Missing attribution? Can't trust it. Score = 0.
- No semantic tags? Can't discover connections. Score = 0.
- No decision logic? Can't automate. Score = 0.

This forces completeness. Partial data has partial value. Complete data has exponential value.

# PART 8: PRACTICAL SCORING WORKFLOW

## 8.1 For Each Piece of Content

1. **Attribution Pass**
   - Source: Who/what? (0-3)
   - Temporal: When? (0-3)
   - Method: How gathered? (0-3)

2. **Authority Pass**
   - Expert tags: Which experts? (list)
   - Authority score: (0-3)

3. **Dimension Pass**
   - Business dimensions: (list from taxonomy)
   - Mechanism dimensions: (list from taxonomy)
   - Density score: (0-3)

4. **Logic Pass**
   - Decision logic present? (0-3)
   - If present, extract TRIGGER/CONDITION/ACTION/OUTCOME

5. **Validation Pass**
   - Current status: (hypothesis → canonical)
   - Evidence: What supports this status?

6. **Multiply**
   - Calculate total multiplier
   - Assign tier (Gold/Silver/Bronze/Raw)

## 8.2 Output Format (YAML Frontmatter)

```yaml
---

# Attribution
```

# PART 9: HOW THIS MULTIPLIES TECHNOLOGY

## 9.1 Qdrant Vector Search + Multiplier

**Without multiplier:** 
- Search returns 10 results
- Human reviews all 10 to assess quality/relevance

**With multiplier:**
- Filter: `multiplier_tier = gold`
- Search returns 10 results, all high-trust
- Can automate response with confidence

## 9.2 Literature-Based Discovery + Multiplier

**Without multiplier:**
- Find pieces with shared keywords
- Many false positives, low signal

**With multiplier:**
- Find pieces with shared semantic dimensions
- Filter by authority_score >= 2
- Only synthesize high-quality sources
- Novel combinations are **attributed pudding recipes**

## 9.3 Agent Automation + Multiplier

**Without multiplier:**
- Agent retrieves any relevant content
- May hallucinate, may cite weak sources

**With multiplier:**
- Agent retrieves content with `has_logic = true`
- Executes pre-validated IF-THEN rules
- Cites expert sources with full attribution
- **Zero hallucination risk for proven content**
