## A1. Local LLM Integration Points

Given the existing stack (iPhone 16, 2x local Llamas, Gwen/Qwen, Obsidian):

### Suggested Architecture
```
┌─────────────────────────────────────────────────────────────┐
│                    ORCHESTRATOR LAYER                        │
│         (Routes requests to appropriate model)               │
└─────────────────────────────────────────────────────────────┘
         │              │              │              │
         ▼              ▼              ▼              ▼
┌─────────────┐ ┌─────────────┐ ┌─────────────┐ ┌─────────────┐
│  Llama A    │ │  Llama B    │ │ Gwen/Qwen   │ │   Claude    │
│ Ops/Systems │ │ Writer/Coach│ │   Cloud     │ │    API      │
│   (Local)   │ │   (Local)   │ │ (Strategy)  │ │ (Fallback)  │
└─────────────┘ └─────────────┘ └─────────────┘ └─────────────┘
```

### Routing Logic
- **Privacy-sensitive tasks** → Local Llama A/B
- **Client data, personal info** → Local Llama A/B
- **Research, web knowledge** → Gwen/Qwen
- **Complex multi-step reasoning** → Gwen/Qwen or Claude
- **Quick transformations, drafts** → Local Llama (offline capable)

## A6. Safety Enhancements

### Suggested Permission Levels (extending spec)
```
Level 0: Observe Only
  - Read emails, calendar, tasks
  - No suggestions shown
  - Passive learning mode

Level 1: Suggest (spec default)
  - Show suggestions and drafts
  - No actions without explicit approval
  - All actions require confirmation

Level 2: Semi-Autonomous (spec)
  - Low-risk actions auto-run (mark as read, file emails)
  - Medium-risk require confirmation
  - High-risk always require confirmation + explanation

Level 3: Trusted (new)
  - User-defined "always allow" actions
  - Time-boxed trust periods ("autonomous for next 2 hours")
  - Automatic revert to Level 2 after period
```

### Undo Enhancements
- 30-second "soft undo" for all actions
- Full audit trail with one-click revert for 7 days
- "Undo all from this session" option
