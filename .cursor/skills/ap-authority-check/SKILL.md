# Authority Congruence Check (Amplified Partners)

**When to use:** Entry signal discipline. During first 60 seconds, if anything causes confusion, signal immediately.

## The Rule

> If anything catches your eye or causes confusion, the best time to mention it is **then** — while eyes are fresh, before normalization sets in.

## Entry Checklist

```
Read AGENTS.md § "Agent session — first 60 seconds"
         ↓
Scan for incongruence:
  - Contradictions between files
  - Outdated terminology
  - Missing status tokens
  - Ambiguous routing
         ↓
Signal immediately if found
```

## What to Signal

| Issue | Token | Action |
|-------|-------|--------|
| Contradiction | [DECISION REQUIRED] | Stop, flag, propose options |
| Missing source | [SOURCE REQUIRED] | Mark, don't treat as truth |
| Incomplete logic | [LOGIC TO BE CONFIRMED] | Bounded options, not invention |
| Confusion | Ask minimal clarifying question | One question beats guessing |

## Authority Files to Check

1. `00_authority/MANIFEST.md` — What's authoritative vs. candidate vs. reference
2. `00_authority/NORTH_STAR.md` — File budget, behavior defaults
3. `AGENTS.md` — Entry point, first 60 seconds
4. `01_truth/processes/` — Routing for work

## The Goal

**Congruence over cleverness.** Incongruence is a hard defect. Do not smooth it into "good vibes."

Signal. Stop. Clarify. Then proceed.
