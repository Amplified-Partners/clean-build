# Project Rules: Knowledge Pipeline

## Architecture
- This is a two-stage Python pipeline: `stage1_label.py` → `stage2_atomise_llm.py`
- Stage 1 exists and works. Focus on Stage 2 (LLM atomisation).
- Read `ATOMISER_SPEC.md` before implementing anything.

## Code Style
- Python 3.10+, use type hints throughout
- Use `pathlib.Path` not `os.path`
- Anthropic SDK for Claude API calls
- YAML frontmatter parsing reuse from `stage1_label.py`
- Chunked file writes for large outputs

## Key Files
- `stage1_label.py` - Reference for frontmatter handling, file iteration
- `stage2_atomise_llm.py` - Main implementation target
- `ATOMISER_SPEC.md` - Requirements specification

## Dependencies
- `anthropic` - Claude API
- Standard library only otherwise (no langchain, no frameworks)

## Testing
- Test with 5-10 files first (`--limit 10`)
- Dry-run mode required (`--dry-run`)
- Log all API calls and costs

## Critical Constraints
- NEVER lose source data
- Always preserve source_path in frontmatter
- Atoms must stand completely alone
- Max 100 words per atom, target 30-60
