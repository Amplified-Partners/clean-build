## Overview

The Invoice Processing API provides a production-ready service for processing invoices with comprehensive validation, automatic expense categorization, and duplicate detection capabilities.

### Key Features
- **Validation**: Required fields, amount ranges, date constraints
- **Categorization**: Automatic expense categorization using keyword matching
- **Duplicate Detection**: Identifies potential duplicates based on vendor, amount, and date
- **Persistence**: SQLite database storage with SQLAlchemy ORM
- **Logging**: Comprehensive logging to file and console

## Endpoints

### POST /process-invoice

Process and store a new invoice with validation, categorization, and duplicate detection.

#### Request

**URL**: `/process-invoice`  
**Method**: `POST`  
**Content-Type**: `application/json`

**Request Body**:
```json
{
  "vendor": "string (required)",
  "amount": number (required, >0),
  "date": "string (required, YYYY-MM-DD, within 90 days)",
  "invoice_number": "string (required)",
  "description": "string (required)"
}
```

**Field Constraints**:
| Field | Type | Required | Constraints |
|-------|------|----------|-------------|
| vendor | string | Yes | Non-empty, no whitespace only |
| amount | number | Yes | Must be > 0 |
| date | string | Yes | Format: YYYY-MM-DD, within last 90 days |
| invoice_number | string | Yes | Non-empty |
| description | string | Yes | Non-empty |

#### Response

**Success Response** (200 OK):
```json
{
  "status": "success",
  "invoice_id": "uuid-string",
  "vendor": "string",
  "amount": number,
  "date": "YYYY-MM-DD",
  "invoice_number": "string",
  "description": "string",
  "category": "Office Supplies|Travel|Professional Services|Equipment|Other",
  "is_duplicate": boolean,
  "processed_at": "ISO-8601 timestamp"
}
```

**Error Responses**:

**400 Bad Request** - Validation Error:
```json
{
  "status": "error",
  "error_type": "validation_error",
  "message": "Invoice validation failed",
  "details": [
    "Amount must be greater than 0",
    "Date must be within last 90 days"
  ]
}
```

**422 Unprocessable Entity** - Missing Required Fields:
```json
{
  "detail": [
    {
      "type": "missing",
      "loc": ["body", "invoice_number"],
      "msg": "Field required",
      "input": {...}
    }
  ]
}
```

**500 Internal Server Error** - Database Error:
```json
{
  "status": "error",
  "error_type": "database_error",
  "message": "An error occurred while processing the invoice",
  "details": ["Database connection failed"]
}
```

#### Examples

**Example 1: Successful Office Supplies Invoice**

Request:
```bash
curl -X POST http://localhost:8000/process-invoice \
  -H "Content-Type: application/json" \
  -d '{
    "vendor": "Office Depot",
    "amount": 150.50,
    "date": "2026-01-10",
    "invoice_number": "INV-2026-001",
    "description": "Office supplies: pens, paper, folders"
  }'
```

Response (200):
```json
{
  "status": "success",
  "invoice_id": "a7adde4a-5279-45d9-9517-93ea028acb5a",
  "vendor": "Office Depot",
  "amount": 150.5,
  "date": "2026-01-10",
  "invoice_number": "INV-2026-001",
  "description": "Office supplies: pens, paper, folders",
  "category": "Office Supplies",
  "is_duplicate": false,
  "processed_at": "2026-01-12T20:16:48.487359Z"
}
```

**Example 2: Validation Error (Negative Amount)**

Request:
```bash
curl -X POST http://localhost:8000/process-invoice \
  -H "Content-Type: application/json" \
  -d '{
    "vendor": "Some Vendor",
    "amount": -50,
    "date": "2026-01-12",
    "invoice_number": "INV-999",
    "description": "Test"
  }'
```

Response (400):
```json
{
  "status": "error",
  "error_type": "validation_error",
  "message": "Invoice validation failed",
  "details": ["Amount must be greater than 0"]
}
```

**Example 3: Python Requests**

```python
import requests

url = "http://localhost:8000/process-invoice"
invoice_data = {
    "vendor": "Travelodge Hotels",
    "amount": 89.99,
    "date": "2026-01-08",
    "invoice_number": "HOTEL-12345",
    "description": "Hotel accommodation, Manchester"
}

response = requests.post(url, json=invoice_data)
print(response.status_code)
print(response.json())
```

### GET /invoice/{invoice_id}

Retrieve a previously processed invoice by its unique ID.

#### Request

**URL**: `/invoice/{invoice_id}`  
**Method**: `GET`  
**Path Parameters**:
- `invoice_id` (string, required): UUID of the invoice

#### Response

**Success Response** (200 OK):
```json
{
  "status": "success",
  "invoice_id": "uuid-string",
  "vendor": "string",
  "amount": number,
  "date": "YYYY-MM-DD",
  "invoice_number": "string",
  "description": "string",
  "category": "string",
  "is_duplicate": boolean,
  "processed_at": "ISO-8601 timestamp"
}
```

**Error Responses**:

**404 Not Found**:
```json
{
  "status": "error",
  "error_type": "not_found",
  "message": "Invoice with ID {invoice_id} not found",
  "details": []
}
```

**500 Internal Server Error**:
```json
{
  "status": "error",
  "error_type": "database_error",
  "message": "An error occurred while retrieving the invoice",
  "details": ["Error message"]
}
```

#### Examples

**Example 1: Successful Retrieval**

Request:
```bash
curl -X GET http://localhost:8000/invoice/a7adde4a-5279-45d9-9517-93ea028acb5a
```

Response (200):
```json
{
  "status": "success",
  "invoice_id": "a7adde4a-5279-45d9-9517-93ea028acb5a",
  "vendor": "Office Depot",
  "amount": 150.5,
  "date": "2026-01-10",
  "invoice_number": "INV-2026-001",
  "description": "Office supplies: pens, paper, folders",
  "category": "Office Supplies",
  "is_duplicate": false,
  "processed_at": "2026-01-12T20:16:48.487359Z"
}
```

**Example 2: Invoice Not Found**

Request:
```bash
curl -X GET http://localhost:8000/invoice/non-existent-id
```

Response (404):
```json
{
  "status": "error",
  "error_type": "not_found",
  "message": "Invoice with ID non-existent-id not found",
  "details": []
}
```

**Example 3: Python Requests**

```python
import requests

invoice_id = "a7adde4a-5279-45d9-9517-93ea028acb5a"
url = f"http://localhost:8000/invoice/{invoice_id}"

response = requests.get(url)
if response.status_code == 200:
    invoice = response.json()
    print(f"Vendor: {invoice['vendor']}")
    print(f"Amount: ${invoice['amount']}")
else:
    print(f"Error: {response.json()['message']}")
```

## Data Models

### Invoice

Represents a processed invoice in the system.

| Field | Type | Description |
|-------|------|-------------|
| invoice_id | string (UUID) | Unique identifier, auto-generated |
| vendor | string | Vendor/supplier name |
| amount | number (float) | Invoice amount in currency |
| date | string (YYYY-MM-DD) | Invoice date |
| invoice_number | string | Vendor's invoice number |
| description | string | Invoice description/details |
| category | string (enum) | Auto-assigned category |
| is_duplicate | boolean | Duplicate flag |
| processed_at | string (ISO-8601) | Processing timestamp |

### Category Enum

Automatically assigned based on keywords in the description:

| Category | Keywords |
|----------|----------|
| **Office Supplies** | office, supplies, stationery, paper, pens, folders |
| **Travel** | taxi, hotel, flight, parking, fuel, accommodation, travelodge |
| **Professional Services** | consulting, accounting, legal, design, professional |
| **Equipment** | computer, equipment, furniture, machinery |
| **Other** | Default if no keywords match |

### Duplicate Detection Rules

An invoice is flagged as duplicate if ALL conditions match:
1. Same vendor (case-insensitive)
2. Same amount (exact match)
3. Same date OR within 24-hour window

**Note**: Duplicates are still processed and stored, only flagged for review.
