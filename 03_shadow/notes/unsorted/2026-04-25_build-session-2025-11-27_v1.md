## Key Decisions Made

### 1. Architecture: Pure Code (No Make.com/n8n)

**Decision:** Migrate from visual workflow builders to pure code architecture.

**Rationale:**
- Infinite scalability, no operation limits
- Git version control (not exported JSON)
- Type safety with compile-time errors
- Proper debugging with stack traces
- Unit/integration testing capability
- Professional team collaboration

**Final Stack:**
| Layer | Tool | Cost/mo |
|-------|------|---------|
| Agent Runtime | Claude Code | ~$100 |
| Workflow Engine | Trigger.dev | ~$29 |
| Tool Integration | MCP | Included |
| Database | PostgreSQL (Railway) | ~$20 |
| Video | HeyGen Pro | $99 |
| Voice/SMS | Twilio | ~$50 |
| Email | Resend | ~$20 |
| **Total** | | **~$356/mo** |

### 2. Build Sequence: C → B

**Decision:** Foundation first, then Success Engine.

**Phase C (Foundation):** PostgreSQL + MCP servers + project structure  
**Phase B (Success Engine):** Trigger.dev jobs + 12-touch nurture + HeyGen video

**Rationale:** Every subsequent feature runs on production-grade infrastructure. No re-work needed.

### 3. Video Included from Day 1

**Decision:** Include HeyGen video in the build despite $99/mo fixed cost.

**Rationale:** Video is a key differentiator. Core value proposition includes personalized video at Touch 5.

### API Routes Completed

All routes now use Prisma for database access:
- GET/POST /api/v1/clients - List/create clients
- GET/PATCH/DELETE /api/v1/clients/{id} - Client operations
- GET /api/v1/clients/{id}/dashboard - Dashboard stats
- GET /api/v1/clients/{id}/leads - List leads with filters
- GET /api/v1/clients/{id}/leads/{id} - Get lead details
- POST /api/v1/clients/{id}/leads/{id}/action - Lead actions
- GET/POST /api/v1/clients/{id}/jobs - List/create jobs
- GET/PATCH /api/v1/clients/{id}/jobs/{id} - Job operations
- POST /api/v1/clients/{id}/jobs/{id}/complete - Complete job

### Webhook Flow Implemented

**POST /api/v1/webhooks/vapi/call-complete**
1. Extract call data from Vapi payload
2. Find client by covered phone number
3. Find/create customer by phone
4. Create lead record
5. Create nurture sequence
6. Trigger Trigger.dev nurture job
7. Notify owner via preferred channels
8. Return success with lead_id
