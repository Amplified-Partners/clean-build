# Hook Model Spec — Habit Formation for Covered AI

## Framework Overview

The Hook Model (Nir Eyal) creates habit-forming products through a four-phase cycle:

```
TRIGGER → ACTION → VARIABLE REWARD → INVESTMENT
    ↑                                      │
    └──────────────────────────────────────┘
                (Loads next trigger)
```

Repeated cycles embed the product into daily routine.

---

## Phase 1: TRIGGER

### External Triggers (Early Stage)

These are explicit prompts that initiate the hook cycle.

| Trigger | Channel | Content | Timing |
|---------|---------|---------|--------|
| New lead alert | WhatsApp | "🚨 New Lead: {{job_type}} in {{postcode}}" | Instant |
| New lead alert | Push | "New enquiry from {{name}}" | Instant |
| Emergency alert | WhatsApp | "🔴 EMERGENCY: {{job_type}} - {{name}}" | Instant |
| Daily digest | Email | "Yesterday: 3 calls, 2 leads, 1 booking" | 8:00 AM |
| Weekly report | Email | "Your week: 18 calls, 9 leads, £2,400 value" | Monday 9:00 AM |
| Job reminder | Push | "Job tomorrow: {{customer}} at {{time}}" | Day before, 6:00 PM |
| Review received | WhatsApp | "⭐ New 5-star review from {{customer}}!" | Instant |

### Internal Triggers (Goal State)

After 2-4 weeks of use, the product should become associated with emotional states:

| Internal Trigger | Emotion/Situation | Product Response |
|-----------------|-------------------|------------------|
| Fear of missing calls | Anxiety | "Covered has it — I can relax" |
| End of workday habit | Routine | "Let me check today's leads" |
| Finishing a job | Completion | "Mark complete, get that review" |
| Boredom between jobs | Idle time | "Check the dashboard for new leads" |
| Phone ringing | Stress | "Gemma's handling it" |

## Phase 2: ACTION

### Fogg Behavior Model: B = MAP

Behavior occurs when **M**otivation, **A**bility, and **P**rompt converge.

| Element | Covered AI Application |
|---------|----------------------|
| **Motivation** | High — it's money (leads = jobs = revenue) |
| **Ability** | Must be effortless — minimize taps |
| **Prompt** | The notification itself |

### Action Optimization

**Goal:** Make actions "stupidly simple"

| Action | Current | Target | How |
|--------|---------|--------|-----|
| View lead details | 2 taps | 1 tap | Deep link in notification |
| Call customer back | 3 taps | 1 tap | Click-to-call in lead view |
| Mark as booked | 3 taps | 1 tap | Quick action button |
| Dismiss lead | 2 taps | 2 taps | Keep as "good friction" |

### Action Flows

**From WhatsApp notification:**
```
[Tap notification]
    ↓
[Lead detail screen loads]
    ↓
[One-tap actions visible:]
    • 📞 Call Back
    • ✅ Book Job
    • ❌ Not Interested
```

**Implementation:**

```typescript
// Deep link structure
const DEEP_LINKS = {
  lead_detail: 'covered://leads/{lead_id}',
  call_customer: 'tel:{phone}',
  quick_book: 'covered://leads/{lead_id}/book',
};

// WhatsApp message with deep link
const whatsappMessage = `
🚨 New Lead: ${lead.jobType}

${lead.customerName}
${lead.postcode}
"${lead.summary}"

Tap to view: ${DEEP_LINKS.lead_detail.replace('{lead_id}', lead.id)}
`;
```

## Phase 3: VARIABLE REWARD

### Why Variability Matters

Fixed rewards become predictable and boring. Variable rewards trigger dopamine release and maintain engagement.

**Example:** Slot machines use variable reward. So should Covered AI.

### Reward Types (Eyal's Framework)

| Type | Description | Covered AI Application |
|------|-------------|----------------------|
| **Tribe** | Social validation | "You're in the top 10% of responders" |
| **Hunt** | Acquiring resources | Unknown job value (£50? £5000?) |
| **Self** | Mastery, completion | "Your best conversion week yet!" |

### Variable Reward Implementation

#### 1. Hunt Rewards (Job Value)

**Key design:** Never show expected job value in notification. Make them tap to find out.

```
❌ Bad: "New Lead: Boiler repair (Est. £200)"
✅ Good: "New Lead: Boiler repair in NE4"
```

The mystery of "how much is this one worth?" drives engagement.

#### 2. Tribe Rewards (Social)

```typescript
// Weekly leaderboard notification (for multi-user accounts)
const tribeRewards = [
  "🏆 You responded faster than 92% of businesses this week",
  "⚡ Your 3-minute average response time beats industry average (47 min)",
  "🌟 Customers rated your service 4.9/5 this month"
];
```

#### 3. Self Rewards (Mastery)

```typescript
// Achievement notifications
const selfRewards = [
  { trigger: 'first_lead', message: "🎉 First lead captured!" },
  { trigger: 'tenth_booking', message: "🔥 10 jobs booked through Covered!" },
  { trigger: 'conversion_rate_up', message: "📈 Your conversion rate hit 75% — your best yet!" },
  { trigger: 'review_milestone', message: "⭐ 25 Google reviews collected!" }
];
```

### Surprise & Delight Moments

Random unexpected rewards increase engagement:

```typescript
// Random positive reinforcement
const surpriseRewards = [
  { 
    condition: 'fast_response', // < 5 min callback
    message: "⚡ Lightning response! Customer loved that."
  },
  {
    condition: 'weekend_booking',
    message: "💪 Working the weekend? That lead was worth £{{value}}!"
  },
  {
    condition: 'emergency_saved',
    message: "🦸 Hero moment! You saved {{customer}} from a {{emergency_type}}."
  }
];
```

## Phase 4: INVESTMENT

### Investment Principle

Users who invest time, data, or effort into a product are more likely to return. Each investment:
1. Loads the next trigger
2. Increases switching costs
3. Improves future rewards

### Investment Actions

| Investment | Next Trigger Loaded | Switching Cost |
|------------|---------------------|----------------|
| Mark lead as "Booked" | Job reminder notification | Job data in system |
| Add job details (date, value) | Calendar sync, completion reminder | Schedule history |
| Complete job | Review request sent | Review automation |
| Customer gets review | Review notification to owner | Reputation building |
| Add notes to customer | Richer future context | Customer history |
| 30+ days of data | Monthly report | Analytics history |
| Customize Gemma's script | Better call handling | Personalization |

### Investment Flow

```
Lead received
    ↓
[View lead] ← ACTION
    ↓
[Variable reward: unknown job value]
    ↓
[Mark as booked] ← INVESTMENT
    ↓
Loads: Job reminder trigger
    ↓
[Complete job] ← INVESTMENT
    ↓
Loads: Review request trigger
    ↓
[Review received] ← VARIABLE REWARD
    ↓
[Check dashboard for stats] ← INVESTMENT
    ↓
Loads: Weekly report trigger
    ↓
... CYCLE CONTINUES ...
```

### Data Lock-In

The more data in the system, the higher the switching cost:

```typescript
// Display in dashboard to reinforce value
```

## Habit Formation Timeline

| Week | Stage | Behavior | Design Focus |
|------|-------|----------|--------------|
| 1 | External triggers | Responds to notifications | Notification quality |
| 2 | Action optimization | Gets faster at processing leads | Reduce friction |
| 3 | Variable reward | Excited to see new leads | Reward variability |
| 4 | Investment forming | Customizes settings, adds data | Investment prompts |
| 5-6 | Association forming | Checks app without prompt | Reduce external triggers |
| 7-8 | Internal triggers | Feels anxiety if not checking | Monitor engagement |
| 8+ | Habit formed | Can't imagine working without it | Maintain & expand |

## Metrics to Track

| Metric | Target | Indicates |
|--------|--------|-----------|
| Notification tap rate | > 80% | Trigger effectiveness |
| Time to action (notification → tap) | < 30 sec | Action simplicity |
| Actions per session | > 2 | Engagement depth |
| Daily active users | > 70% | Habit formation |
| 7-day retention | > 85% | Early habit |
| 30-day retention | > 70% | Established habit |
| NPS | > 50 | Reward satisfaction |

---

## Anti-Patterns to Avoid

| Anti-Pattern | Why It's Bad | Instead |
|--------------|--------------|---------|
| Too many notifications | Alert fatigue, user disables | Batch non-urgent, prioritize |
| Predictable rewards | Boring, disengaging | Add variability |
| Complex actions | Friction kills habit | Simplify ruthlessly |
| No investment prompts | No lock-in, easy to churn | Add low-friction investments |
| Guilt-based triggers | Negative association | Positive, opportunity-focused |

## Code Implementation Summary

### Trigger System

```typescript
// src/services/trigger-service.ts

export async function sendTrigger(
  client: Client,
  triggerType: TriggerType,
  data: TriggerData
) {
  const config = TRIGGER_CONFIG[triggerType];
  
  // Select channel based on priority and user preferences
  const channel = selectChannel(client, config);
  
  // Format message
  const message = formatTriggerMessage(config.template, data);
  
  // Add deep link for action
  const deepLink = generateDeepLink(triggerType, data);
  
  // Send via appropriate service
  await sendNotification(channel, client, message, deepLink);
  
  // Track for optimization
  await trackTrigger(client.id, triggerType, channel);
}
```

### Variable Reward Engine

```typescript
// src/services/reward-service.ts

export async function checkAndSendReward(
  client: Client,
  event: RewardEvent
) {
  // Check for milestone achievements
  const milestones = await checkMilestones(client.id);
  
  // Check for surprise rewards (random, 10% chance)
  const surprise = Math.random() < 0.1 
    ? selectSurpriseReward(event) 
    : null;
  
  // Send if earned
  if (milestones.length > 0 || surprise) {
    await sendRewardNotification(client, milestones, surprise);
  }
}
```

### Investment Tracker

```typescript
// src/services/investment-service.ts

export async function trackInvestment(
  client: Client,
  investmentType: InvestmentType,
  data: any
) {
  // Record investment
  await recordInvestment(client.id, investmentType, data);
  
  // Load next trigger
  const nextTrigger = INVESTMENT_TRIGGER_MAP[investmentType];
  if (nextTrigger) {
    await scheduleNextTrigger(client, nextTrigger, data);
  }
  
  // Update switching cost metrics
  await updateSwitchingCost(client.id);
}
```
