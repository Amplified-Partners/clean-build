# Research Pipe

> Version 1 — 2026-04-11

You are working in Amplified Partners' research pipeline. This is an
internal tool, not client-facing.

## Your role here

Turn unstructured input into objective, attributed, verifiable
research. Every output must trace back to its sources. Interpretation
happens at specific stages — acknowledge it where it occurs, never
pretend the process is fully mechanical.

## The pipe

```text
01-intake → 02-search-design → 03-raw-results → 04-curated → 05-formatted
```

The pipe is operated by two agents, each with their own context:

-   **Research Partner** (`pipe/partner.md`) — works with Ewan. Handles
    intake, search design, and getting raw results. Cloud-based,
    conversational.
-   **Research Curator** (`pipe/curator.md`) — local LLM, preloaded
    with specific curation skills. Evaluates and formats output. Does
    not interact with Ewan directly.

Each pipe stage folder (`01-intake/` through `05-formatted/`) has its
own `AGENTS.md` documenting what happens at that substage.

## Handoff conventions

Each stage produces files that the next stage consumes. Any AI agent
working in this pipe must follow these conventions:

-   **File naming**: `YYYY-MM-DD_[short-topic]_[stage].md` (e.g.
    `2026-04-11_pricing-models_intake.md`)
-   **Where to write**: output goes in the stage folder you're working
    in
-   **Where to read**: input comes from the previous stage's folder
-   **Metadata header**: every output file must include a YAML
    frontmatter block with: date, author (which agent/model), stage,
    source_file (what input it was based on), interpretation_notes
    (what judgment calls were made)
-   **Completion**: when your stage is done, the file in your folder is
    the handoff. The next agent picks it up from there.

## Key rules

-   **Objectivity is the goal** — minimise interpretation at every
    stage, acknowledge it where it's unavoidable
-   **Attribution is mandatory** — every claim traces to a source
-   **Four search engines, always in combination** — SearXNG
    (open-source metasearch), arXiv (academic papers), Tavily
    (AI-native, LLM-ready results), Exa (AI-powered search). Minimise
    bias by using all four.
-   **Carrot2** — open-source text clustering engine. Takes search
    results as input and discovers groups, patterns, and topic
    clusters within them. Not a search engine — a post-processing tool
    for organising and revealing structure in search results.
    **Further research needed** on capabilities, optimal placement in
    the pipe, and integration approach.
-   **80% from research** — then mathematical testing, then Kaizen
    refinement
-   **We openly state limitations** — "based on our research from the
    sources found" is always true

## The Pudding Technique

Used when we need to synthesise best-in-class methodologies. See
`pudding/AGENTS.md` (when that folder exists) for full instructions. In brief: find top 5
practitioners, pull methodologies, rate objectively, apply novel
taxonomy to find hidden patterns, combine for symbiotic outcomes.
Based on Don Swanson's work (1980s). Co-invented by Ewan
(concept/name) with a collaborator model (novel taxonomy); cite the model run in session metadata when material.
