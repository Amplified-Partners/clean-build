# Landing Page & Website Framework

A practical, prioritized system for building high-converting pages.

---

## The Core Model: B=MAP

Every conversion requires three things happening simultaneously:

```
Behavior = Motivation × Ability × Prompt
```

| Element | What It Means | Your Control Level |
|---------|---------------|-------------------|
| **Motivation** | Why they want it | Low (they arrive with this) |
| **Ability** | How easy it is | HIGH (this is your job) |
| **Prompt** | The trigger to act now | High (CTAs, urgency) |

**Key insight:** Don't waste effort increasing motivation. Focus on reducing friction (Ability) and providing clear prompts.

---

## Page Structure: The 7-Section Blueprint

Every landing page follows this sequence. Not all sections required for every page - but this is the order.

### Section 1: Hero (Above the Fold)
**Job:** Answer "What is this and why should I care?" in 5 seconds.

| Element | Purpose | Requirements |
|---------|---------|--------------|
| **Headline** | State the transformation | 6-12 words, benefit-focused, no jargon |
| **Subheadline** | Explain how | 15-25 words, clarifies the mechanism |
| **Hero image/video** | Show the outcome | Real, not stock. Show the "after" state |
| **Primary CTA** | Tell them what to do | Action verb + outcome. One CTA only. |
| **Trust snippet** | Reduce immediate skepticism | "500+ businesses" or one logo row |

**Test:** Cover everything below the fold. Can someone understand and act? If no, rewrite.

```
┌─────────────────────────────────────────┐
│ Logo                    Nav (minimal)   │
├─────────────────────────────────────────┤
│                                         │
│   [Headline - The Transformation]       │
│   [Subheadline - How It Works]          │
│                                         │
│   [ Primary CTA Button ]                │
│                                         │
│   ★★★★★ "Quote from customer" - Name    │
│                                         │
│              [Hero Image]               │
│                                         │
└─────────────────────────────────────────┘
```

**Hero Formulas That Work:**

| Formula | Example |
|---------|---------|
| [Outcome] without [Pain] | "Never miss a customer call without hiring staff" |
| [Audience]: [Outcome] | "Tradespeople: Get paid faster, chase less" |
| The [Adjective] way to [Outcome] | "The simple way to answer every call" |
| Stop [Pain]. Start [Outcome]. | "Stop losing leads. Start growing." |

---

### Section 2: Problem Agitation
**Job:** Make them feel understood. Articulate their pain better than they can.

- 3-5 bullet points of specific pain
- Use their language (not yours)
- End with implied question: "Sound familiar?"

```
## Tired of...

- Missing calls while you're on a job?
- Losing customers to competitors who answer faster?
- Paying for a receptionist who calls in sick?
- Checking voicemails at 9pm?
```

**Rule:** If you can't write this section from memory, you don't know your customer well enough.

---

### Section 3: Solution Introduction
**Job:** Bridge from problem to your product. Position yourself as the guide, not the hero.

- Introduce your solution in one sentence
- Explain the mechanism (how it works) in 3 steps max
- Keep it simple - no feature dumps

```
## Here's how it works

1. **Forward your calls** → Takes 2 minutes to set up
2. **Gemma answers** → 24/7, sounds human, knows your business  
3. **You get the details** → Jobs booked, emergencies flagged, leads captured
```

**Pattern:** [Simple action] → [What happens] → [Outcome for them]

---

### Section 4: Social Proof
**Job:** Prove it works for people like them.

**Hierarchy of proof (strongest to weakest):**

1. **Video testimonials** - Real person, real story
2. **Case study with numbers** - "Ralph went from 3 missed calls/day to zero"
3. **Written testimonial with photo** - Name, business, location
4. **Review aggregation** - "4.9 stars from 127 reviews"
5. **Logos/badges** - Known brands, certifications
6. **Numbers** - "2,847 businesses use Covered"

**Minimum viable proof:**
- 1 detailed testimonial (with photo, name, business type)
- 1 metric ("saves 10 hours/week" or "answers 500 calls/month")
- Trust badges (payment providers, security)

```
┌─────────────────────────────────────────┐
│  "Gemma's paid for herself ten times   │
│   over. I was skeptical about AI but   │
│   customers can't tell the difference." │
│                                         │
│   [Photo] Ralph, Titan Plumbing         │
│           Newcastle                     │
└─────────────────────────────────────────┘
```

---

### Section 5: Features → Benefits
**Job:** Show what they get, framed as outcomes.

**Never list features. Always translate to benefits:**

| ❌ Feature | ✅ Benefit |
|-----------|-----------|
| 24/7 availability | Never miss a 2am emergency call |
| AI-powered | Answers like a real receptionist |
| SMS notifications | Know instantly when someone calls |
| Call recording | Replay conversations you missed |
| CRM integration | Customer details, ready when you call back |

**Format options:**
- Icon grid (3-4 items per row)
- Alternating image/text sections
- Comparison table (you vs. alternatives)

---

### Section 6: Objection Handling
**Job:** Address the reasons they haven't bought yet.

**Common objections by type:**

| Objection Type | Example | How to Handle |
|----------------|---------|---------------|
| **Trust** | "Is this legit?" | Testimonials, guarantees, about page |
| **Price** | "Is it worth it?" | ROI calculator, comparison to alternatives |
| **Effort** | "Is it hard to set up?" | "2 minutes to start" + demo video |
| **Timing** | "Do I need this now?" | Urgency, cost of waiting |
| **Fit** | "Will it work for me?" | Specific use cases, vertical proof |

**Formats:**
- FAQ accordion
- "Will it work for..." section with use cases
- Comparison table vs. alternatives
- Risk reversal (guarantee, free trial)

```
## Common questions

**Will customers know it's AI?**
Most can't tell. Gemma uses a natural Northern accent and 
handles conversations like a real receptionist. Listen to 
a sample call → [Audio player]

**What if there's a real emergency?**
Gemma recognises emergencies and forwards them to your 
mobile immediately. You set the rules for what counts.

**Can I try before I commit?**
Yes. Call 0800 268 3733 right now and Gemma will answer 
as if she's your receptionist. No signup needed.
```

---

### Section 7: Final CTA
**Job:** Make the ask. Reduce friction to zero.

**Elements:**
- Repeat the core value proposition (condensed)
- Clear CTA button (same as hero)
- Friction reducer: "No credit card required" / "Cancel anytime"
- Urgency if genuine: "First 50 get free onboarding"

```
┌─────────────────────────────────────────┐
│                                         │
│   Ready to stop missing calls?          │
│                                         │
│   [ Start Your Free Trial ]             │
│                                         │
│   No credit card needed. Setup in 2min. │
│                                         │
└─────────────────────────────────────────┘
```

## CTA Rules

### Copy Formulas

| Pattern | Example | Use When |
|---------|---------|----------|
| **Verb + Outcome** | "Start Saving Time" | Primary CTA |
| **Get + Thing** | "Get Your Demo" | Lead gen |
| **Verb + Possessive** | "Start My Free Trial" | Increases ownership |
| **Simple Action** | "Sign Up" / "Buy Now" | When action is obvious |

### Design Rules

- **One primary CTA per screen** - Don't compete with yourself
- **Contrast** - Button must be the most visually prominent element
- **Size** - Minimum 44px height for mobile tap targets
- **Placement** - Above fold + end of each major section
- **Repeat** - Same CTA, same words, throughout page

### Friction Reducers (place near CTA)

- "No credit card required"
- "Cancel anytime"  
- "2-minute setup"
- "30-day money-back guarantee"
- "Free for 14 days"
- "No commitment"

## Trust Signal Checklist

### Mandatory (every page)

- [ ] SSL certificate (HTTPS)
- [ ] Contact information visible (email minimum, phone better)
- [ ] Privacy policy link in footer
- [ ] Terms of service link in footer
- [ ] Real company name and address

### High Impact (landing pages)

- [ ] Customer testimonial with photo and name
- [ ] Trust badges (payment providers, security)
- [ ] Social proof number ("X customers" or "X reviews")
- [ ] Media mentions / "As seen in" (if applicable)
- [ ] Money-back guarantee or risk reversal

### Nice to Have

- [ ] Team photos on about page
- [ ] Video testimonials
- [ ] Live chat
- [ ] Case studies with metrics
- [ ] Third-party review badges (Trustpilot, G2)

## Mobile-First Checklist

### Layout
- [ ] Single column on mobile
- [ ] CTA visible without scrolling
- [ ] Tap targets minimum 44px
- [ ] Thumb-friendly navigation

### Performance  
- [ ] LCP under 2.5 seconds
- [ ] Images optimized (WebP, lazy loading)
- [ ] No layout shift (CLS near 0)
- [ ] Works on 3G connection

### Content
- [ ] Headlines readable without zooming
- [ ] Forms use appropriate input types (tel, email)
- [ ] No horizontal scrolling
- [ ] Click-to-call on phone numbers

## Copy Principles

### Clarity Over Cleverness

| ❌ Clever | ✅ Clear |
|----------|---------|
| "Revolutionize your workflow" | "Get more done in less time" |
| "Unlock your potential" | "Learn new skills" |
| "Synergize your operations" | "Work better together" |
| "Explore our solutions" | "See pricing" |

### Specificity Wins

| ❌ Vague | ✅ Specific |
|---------|-----------|
| "Save time" | "Save 10 hours per week" |
| "Affordable" | "From £9/month" |
| "Fast setup" | "Ready in 2 minutes" |
| "Many customers" | "2,847 businesses" |
| "Great support" | "Response within 1 hour" |

### Voice Guidelines

- **Use "you" and "your"** - Not "our customers" or "users"
- **Active voice** - "Gemma answers your calls" not "Calls are answered by Gemma"
- **Present tense** - "You get" not "You will get"
- **Short sentences** - 15 words max where possible
- **One idea per paragraph** - Break up long blocks

## Urgency & Scarcity (Use Honestly)

### Legitimate Urgency

| Tactic | Example | When It's Real |
|--------|---------|----------------|
| **Limited time** | "Offer ends Friday" | Actual deadline exists |
| **Limited quantity** | "Only 12 pilot slots left" | Actually limited |
| **Price increase** | "Price goes up Jan 1" | You're actually raising prices |
| **Seasonal** | "Get ready for summer" | Genuinely time-sensitive |

### Fake Urgency (Never Use)

- Countdown timers that reset
- "Only 3 left!" when there's infinite stock
- Fake "limited time" offers that run forever
- Manufactured scarcity

**Rule:** If a customer discovers the urgency is fake, you've lost them forever.

## Testing Priorities

When you have traffic, test in this order (highest impact first):

### 1. Headlines
Test: Different value propositions, benefit angles, specificity levels

### 2. CTA Copy
Test: Verb choices, outcome framing, first person vs. second person

### 3. Social Proof Placement
Test: Above fold vs. below, type of proof, quantity of testimonials

### 4. Form Length
Test: Number of fields, multi-step vs. single page, required vs. optional

### 5. Page Length  
Test: Long-form vs. short-form (depends on price point and complexity)

### What NOT to Test Early
- Button colors (minimal impact)
- Font choices (almost no impact)
- Minor copy tweaks (test big changes first)

## Page Types Quick Reference

### Landing Page (Ads/Campaigns)
- Single focus, one CTA
- No navigation (or minimal)
- Matches ad message exactly
- 7-section structure above

### Homepage
- Multiple audiences possible
- Clear paths for each
- Navigation present
- Hero + 3-4 key sections

### Product Page
- All details available
- Multiple CTAs okay (add to cart, learn more)
- Full navigation
- Heavy on proof and specifications

### Pricing Page
- Easy comparison
- Most popular plan highlighted
- FAQ for objections
- Clear CTA per tier

## Pre-Launch Checklist

### Content
- [ ] Headline passes 5-second test
- [ ] Subheadline explains mechanism
- [ ] Benefits (not features) are clear
- [ ] At least one testimonial
- [ ] All copy proofread
- [ ] CTA copy is action-oriented

### Technical
- [ ] Page loads under 3 seconds
- [ ] Mobile responsive
- [ ] All links work
- [ ] Forms submit correctly
- [ ] Analytics installed
- [ ] Conversion tracking set up

### Trust
- [ ] SSL active
- [ ] Contact info visible
- [ ] Privacy policy linked
- [ ] Testimonials are real
- [ ] Guarantees clearly stated

### SEO (if applicable)
- [ ] Title tag set (50-60 chars)
- [ ] Meta description set (150-160 chars)
- [ ] H1 present and relevant
- [ ] Images have alt text
- [ ] URL is clean and descriptive

## Quick Wins (Do These First)

If you only have an hour, do these:

1. **Rewrite your headline** to state the transformation
2. **Add one real testimonial** with photo and name
3. **Make CTA visible** above the fold
4. **Add friction reducer** text below CTA
5. **Remove one form field** you don't actually need
