## Brand Overview

covered.AI is a low-friction optimization service for UK SMBs in the £500K-£3M revenue range. We tighten processes without disrupting operations. The name combines "covered" (protection, reliability, safety net) with "AI" (automation, intelligence, background operation). We are the background optimizer—no transformation, no disruption, just results.

Our founder, Ewan Bramley, brings 30 years of dental practice experience from Byker, Newcastle upon Tyne. This isn't a tech startup built by people who've never run a business. It's a service built by someone who understood the same challenges UK service business owners face daily: 60+ hour weeks, late payments, admin overwhelm, and the constant feeling that the business can't run without you. We don't fix what's broken. We tighten what already works and automate the busywork so you can focus on what actually makes money.

**Positioning Statement:** covered.AI helps UK service SMBs (£500K-£3M, 3-15 staff) recover 10-20 hours per month and collect £8K-15K more per month through background automation—invoicing, follow-ups, and process documentation—without disrupting how they work.

## Logo Usage

### Primary Logo

The primary logo consists of the covered.AI wordmark with the icon positioned to the left. This is the preferred logo for all applications where space permits.

**File Specifications:**
- Primary logo: `covered-ai-primary-logo.svg` (vector, infinitely scalable)
- Primary logo PNG: `covered-ai-primary-logo-512.png` (512x preferred, also 1024, 2048)
- Primary logo PDF: `covered-ai-primary-logo.pdf` (print applications)

**Clear Space Rules:**

The minimum clear space around the logo is defined by the height of the "A" in covered. Ensure no other elements (text, images, graphics, page edges) intrude into this space.

```
[Space] ICON [Space] WORDMARK [Space]
```

**Minimum Sizes:**

| Application | Minimum Width |
|-------------|---------------|
| Print | 25mm (1 inch) |
| Digital/Desktop | 120px |
| Mobile/App Icon | 44px |
| Favicon | 16px |

### Icon-Only Logo

The icon may be used standalone when the full logo won't fit or when the brand is already established in context. This is appropriate for app icons, favicons, social media profile pictures, and small print applications.

**File Specifications:**
- Icon-only SVG: `covered-ai-icon-only.svg`
- Icon-only PNG: `covered-ai-icon-only-512.png` (512x512)
- Favicon: `covered-ai-favicon.ico` (16x16, 32x32, 48x48)

**When to Use Icon-Only:**
- App icons and mobile applications
- Favicon in browser tabs
- Social media profile pictures (LinkedIn, Twitter, Instagram)
- Print items smaller than minimum primary logo size
- When the brand is already strongly established in context

### Wordmark-Only Logo

The wordmark may be used alone when the icon is not needed or when vertical space is limited. This is appropriate for merchandise, signage with height restrictions, and certain digital applications.

**File Specifications:**
- Wordmark-only SVG: `covered-ai-wordmark-only.svg`
- Wordmark-only PNG: `covered-ai-wordmark-only-512.png`

**When to Use Wordmark-Only:**
- Merchandise (t-shirts, mugs, pens)
- Signage with vertical space restrictions
- Letterhead and formal documents
- When the icon would be visually distracting

### Reversed Logo (Dark Backgrounds)

For dark backgrounds, use the reversed (white) version of the logo to ensure proper contrast and readability.

**File Specifications:**
- Reversed primary: `covered-ai-primary-logo-reversed.svg`
- Reversed icon: `covered-ai-icon-only-reversed.svg`
- Reversed wordmark: `covered-ai-wordmark-only-reversed.svg`

**When to Use Reversed Logo:**
- Dark backgrounds (#0A2540 or darker)
- Photography with dark overlay
- Video backgrounds
- Dark-colored merchandise

### Logo Never-Do List

**Never do the following with the logo:**

- ❌ Stretch, compress, or distort the logo proportions
- ❌ Rotate the logo (except app icon variants)
- ❌ Change the colors of any logo element
- ❌ Add drop shadows, glows, or effects
- ❌ Place the logo on busy or low-contrast backgrounds
- ❌ Separate the icon from the wordmark (except approved variations)
- ❌ Add outlines or borders around the logo
- ❌ Use the logo smaller than minimum sizes
- ❌ Replace the typeface with any other font
- ❌ Create custom logo variations without approval

| Color Name | Hex | RGB | Usage |
|------------|-----|-----|-------|
| White | #FFFFFF | 255, 255, 255 | Backgrounds, reverse text |
| Off-White | #F0F4F8 | 240, 248, 248 | Alternative Gray | #1 backgrounds |
| DarkA202C | 26, 32, 42 | Footer backgrounds |
| Muted Teal | #00B89C | 0, 184, 156 | Hover states |

### Color Usage Rules

**Primary Color Usage:**
- Navy Blue for H1 headings, navigation, footer backgrounds
- Bright Teal for primary CTA buttons, links, highlights
- Navy text on Light Gray backgrounds (WCAG AA compliant)

**Text Color Usage:**
- Charcoal Gray (#2D3748) for body text on white backgrounds
- White (#FFFFFF) for text on Navy Blue backgrounds
- Navy Blue (#0A2540) for headings on white backgrounds
- Avoid using Light Gray for text (insufficient contrast)

**Background Color Usage:**
- Light Gray (#F7FAFC) for section backgrounds
- White (#FFFFFF) for content cards and containers
- Navy Blue (#0A2540) for footer and hero sections

### Accessibility Contrast Ratios

All color combinations must meet WCAG AA standards (minimum 4.5:1 for body text, 3:1 for large text).

| Combination | Contrast Ratio | Pass/Fail |
|-------------|----------------|-----------|
| Charcoal on White | 10.6:1 | ✅ Pass (AAA) |
| Navy on White | 14.9:1 | ✅ Pass (AAA) |
| Teal on White | 3.2:1 | ⚠️ Large text only |
| White on Navy | 14.9:1 | ✅ Pass (AAA) |
| Teal on Navy | 2.9:1 | ❌ Fail (not for text) |

**Text Size Adjustments:**
- For text smaller than 18px (or 14px bold), always use Charcoal on White or White on Navy
- For large text (18px+ or 14px+ bold), Teal on White is acceptable for headings

## Typography

### Font Pairings

**Headings: Satoshi Bold**

Satoshi is a modern, geometric sans-serif typeface that conveys confidence and professionalism. Use Satoshi Bold for all headings (H1-H6) and where emphasis is needed.

**Primary Heading Weights:**
- H1: Satoshi Bold, 48px (desktop), 32px (mobile)
- H2: Satoshi Bold, 36px (desktop), 28px (mobile)
- H3: Satoshi Bold, 28px (desktop), 24px (mobile)
- H4: Satoshi Bold, 22px
- H5: Satoshi Bold, 18px
- H6: Satoshi Bold, 16px

**Body: Inter Regular**

Inter is a highly readable, open-source sans-serif designed for screens. Use Inter Regular for all body text, paragraphs, and UI elements.

**Body Weights:**
- Body regular: Inter Regular, 16px
- Body small: Inter Regular, 14px
- Body large: Inter Regular, 18px

**Monospace: JetBrains Mono**

JetBrains Mono is a readable monospace font for data, code, metrics, and technical content.

**Monospace Usage:**
- Data points, statistics
- Code snippets
- Email addresses
- Metrics and numbers

### Typography Guidelines

**Paragraph Spacing:**
- Space between paragraphs: 24px (1.5em)
- First paragraph: No top margin
- Last paragraph: No bottom margin

**Heading Spacing:**
- Space before headings: 48px
- Space after headings: 16px
- Headings directly following headings: 32px

**Text Alignment:**
- Default: Left-aligned
- Center-aligned: Hero sections, CTAs, quotes
- Right-aligned: Only when layout requires (avoid if possible)

**Character Limits:**
- Line length: 60-75 characters optimal
- Line length max: 80 characters (CSS: max-width: 65ch)
- Paragraph width: 700px maximum

## Voice and Tone

### Voice Principles

covered.AI speaks with clarity, honesty, and respect. We don't use corporate jargon, salesy hype, or vague promises. We use specific numbers, admit limitations, and treat business owners as intelligent adults who've built successful companies without our help.

**Core Voice Characteristics:**
- **Straightforward:** No buzzwords, no filler, no corporate speak
- **Specific:** Numbers, timelines, outcomes—not vague adjectives
- **Honest:** Admit what we don't do, set realistic expectations
- **Human:** Conversational but professional, never casual to the point of unprofessional
- **Respectful:** These owners have built businesses. We help them run them better.

### Tone Guidelines

**DO use this tone:**

- "We automate the boring stuff so you can focus on what actually makes money."
- "Typical setup: 4-6 weeks. Disruption: Zero."
- "You don't need transformation. You need optimization."
- Specific numbers: "£8K-15K more per month," "10-20 hours back per week"
- Honest limitations: "We won't transform your business overnight. We tighten what works."
- Conversational: "Here's what happens next," "Quick question," "Tell me about..."

**DON'T use this tone:**

- Corporate jargon: "synergize," "leverage," "optimize operational efficiencies"
- Salesy hype: "revolutionary," "game-changing," "transformational"
- Vague promises: "increase efficiency," "improve productivity," "boost growth"
- Condescension: Explaining basic business concepts to owners
- Overcomplication: Technical details owners don't need

### Vocabulary Guidelines

**Preferred Terms (Use These):**

| Avoid | Use Instead |
|-------|-------------|
| leverage | use, apply |
| synergize | combine, work together |
| transform | improve, tighten, optimize |
| revolutionize | change, improve significantly |
| cutting-edge | modern, current |
| end-to-end | complete, full |
| best-in-class | excellent, proven |
| world-class | excellent, high-quality |
| disruption | change, impact |
| paradigm shift | significant change |
| ecosystem | system, network |
| onboard | integrate, set up |

**Words to Avoid Entirely:**

- revolutionary
- game-changing
- transformational
- disruptive
- innovative (use "modern" or "current" if needed)
- agile (unless actually talking about agile methodology)
- synergistic
- best-in-class
- next-level
- bleeding-edge

**Numbers and Statistics:**

Always use specific numbers instead of vague descriptors:

- ✅ "£8K-15K more collected per month"
- ✅ "10-20 hours saved per week"
- ✅ "30% more leads captured"
- ✅ "90-day implementation"
- ❌ "significant savings"
- ❌ "substantial time back"
- ❌ "dramatic improvement"

### Messaging Framework for Pain Points

**Problem Acknowledgment:**

Before offering solutions, validate the prospect's current situation:

- "You're not broken. Your business works. It just takes too much of your time."
- "Most consultants want to 'transform' your business. You just need it to work better."
- "The problem isn't your team or your ideas. It's that too much runs through you."

**Solution Positioning:**

Frame solutions as tightening, not transforming:

- "We tighten your processes. You keep running your business."
- "We automate the busywork. You focus on what actually makes money."
- "We work in the background. Your team won't even notice."

**Results Framing:**

Always link to specific, measurable outcomes:

- "Typical clients get 10-20 hours back per week."
- "Most collect £8K-15K more per month from late payments."
- "Setup takes 4-6 weeks. Disruption: zero."

## Visual Elements

### Imagery Style

**Photography Guidelines:**

covered.AI imagery should feel authentic, professional, and relatable. We avoid stock photography clichés and overly polished corporate imagery.

**Photography Characteristics:**
- Real UK business environments (offices, practices, shops)
- Real people (business owners, staff, not models)
- Natural lighting preferred over artificial
- Action shots showing work in progress
- Avoid: Handshakes, podium speeches, team jumping, glass office towers

**Image Categories:**

1. **Business Owner Portraits:** 40-55 year old, professional but approachable, in their work environment
2. **Workspace Shots:** Clean, organized workspaces showing actual work
3. **Process Documentation:** Screenshots, workflows, automation in action
4. **Results Visualization:** Charts, metrics, before/after comparisons

**Stock Photography Sources:**
- Unsplash (preferred for authentic look)
- Pexels
- Burst (Shopify)
- Custom photography when possible

### Icon Library

**Icon Style:**

Simple, outline-style icons with consistent stroke width (2px). Icons should be clear, recognizable, and not overly detailed.

**Icon Categories:**

1. **Process Icons:**
   - Audit/document icon
   - Automation/gear icon
   - Background/run icon
   - Results/chart icon

2. **Industry Icons:**
   - Dental practice icon
   - Salon icon
   - Vet clinic icon
   - Trade business icon

3. **Feature Icons:**
   - Invoice/payment icon
   - Calendar/schedule icon
   - Email/communication icon
   - Data/analytics icon

**Icon Resources:**
- Phosphor Icons (outline style, consistent weights)
- Heroicons (clean, recognizable)
- Lucide (modern, simple)

**Icon Usage:**
- Minimum size: 24px
- Stroke weight: 2px
- Color: Navy Blue (#0A2540) or Charcoal (#2D3748)
- Corner radius: 4px (consistent)

### Photography Treatment

**Backgrounds:**
- Light Gray (#F7FAFC) for section backgrounds
- White (#FFFFFF) for content cards
- Navy Blue (#0A2540) for hero sections and footers

**Overlays:**
- Use subtle gradients only when text needs to overlay images
- Gradient: Navy Blue to transparent (90% to 0%)
- Maximum opacity: 70%

**Image Borders:**
- No borders on images by default
- Subtle shadow on cards: `box-shadow: 0 2px 8px rgba(0, 0, 0, 0.08)`
- Rounded corners: 8px maximum

## Analytics and Tracking

### UTM Parameters

All marketing URLs should include UTM parameters for tracking:

```
https://covered.ai/[page]?
utm_source=[source]&
utm_medium=[medium]&
utm_campaign=[campaign]&
utm_content=[content]&
utm_term=[term]
```

**UTM Values:**

| Parameter | Examples |
|-----------|----------|
| utm_source | linkedin, google, email, twitter, newsletter |
| utm_medium | social, cpc, email, organic, referral |
| utm_campaign | launch, blog-post, case-study, webinar |
| utm_content | linkedin-post-1, email-welcome, cta-button |
| utm_term | covered-ai, process-optimization |

**Example:**
```
https://covered.ai/how-it-works?utm_source=linkedin&utm_medium=social&utm_campaign=launch&utm_content=linkedin-post-1
```
