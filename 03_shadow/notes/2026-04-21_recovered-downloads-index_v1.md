## Practical Read

At the moment, this does **not** look like the files have vanished.

It looks more like one of these:

1. files arrive in bursts and are easy to lose in the noise of `Downloads`
2. some files are still being written by Chrome and can look unstable mid-download
3. the folder is acting as a drop zone, not a stable holding area

## Next Safe Move

If this keeps happening, the right fix is:

- keep dropping into `Downloads`
- but move or copy priority files into a dated recovery folder or authority folder the same night
- maintain an index like this after each batch
