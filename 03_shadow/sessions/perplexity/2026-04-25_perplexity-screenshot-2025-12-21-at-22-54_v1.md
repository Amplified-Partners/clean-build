FWIW, I found that once I added a core rule to make sure a file was always read before calling a tool to edit in case of recent edits, I saw a lot more success and a lot less issues
## Brief overview
- Critical rule: always refresh file content before any edit operation to prevent AI model calling/routing issues
- Prevents conflicts from editing stale file content across different AI models and tools
- Ensures edit operations apply to current file state, not outdated content
- Eliminates sync errors that cause model switching failures and content corruption
